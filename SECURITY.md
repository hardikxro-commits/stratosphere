# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Stratos, please report it privately.

**Do not** open a public GitHub issue.

### How to report

- Email: [hardikxro-commits@users.noreply.github.com](mailto:hardikxro-commits@users.noreply.github.com)
- Or open a [private vulnerability report](https://github.com/hardikxro-commits/stratosphere/security/advisories/new)

We'll acknowledge receipt within 48 hours and provide an estimated timeline for a fix.

## Scope

We take the following seriously:

- Exposure of user data or session tokens
- Injection attacks (XSS, SQLi, etc.)
- Authentication bypass
- Sensitive information exposure in source or builds
- Dependency vulnerabilities with active exploits

## Out of Scope

- Missing HTTPS on subdomains we don't control
- Rate limiting on non-critical endpoints
- Social engineering attacks
- Theoretical vulnerabilities without a practical exploit

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅ Yes    |
| Older   | ❌ No     |

## Preferred Format

Include:
- Type of vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We'll credit you in release notes once resolved.
